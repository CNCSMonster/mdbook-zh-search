# mdbook-zh-search 架构梳理

## 📋 当前搜索架构

### 核心问题：搜索方式是否分为前后端？

**答案：是的，分为前后端，但都在浏览器端运行！**

---

## 🏗️ 架构总览

```
┌─────────────────────────────────────────────────────────────┐
│                    构建时 (Build Time)                       │
│                   (Rust 代码，本地运行)                       │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  mdbook-zh-search (修改后的 mdbook)                  │   │
│  │                                                      │   │
│  │  1. 读取 Markdown 文件                                │   │
│  │  2. 中文分词 (jieba-rs)                              │   │
│  │  3. 构建索引 (elasticlunr-rs)                        │   │
│  │  4. 生成 searchindex-xxxx.js                         │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              ↓
                         输出文件
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                   运行时 (Runtime)                           │
│                 (JavaScript，浏览器端运行)                    │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  浏览器加载生成的 HTML 书籍                             │   │
│  │                                                      │   │
│  │  1. 加载 searchindex-xxxx.js (索引数据)              │   │
│  │  2. 加载 elasticlunr.min.js (搜索库)                 │   │
│  │  3. 加载 searcher.js (搜索 UI 逻辑)                    │   │
│  │  4. 用户搜索 → 浏览器端查找索引 → 显示结果            │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## 🔍 关键澄清

### ❌ 常见误解

```
误解 1: "后端搜索" = 服务器端搜索
❌ 错误！mdbook 是静态站点生成器，没有服务器！

误解 2: "前端搜索" = 只在用户输入时搜索
❌ 错误！索引是预先生成的，搜索在浏览器端进行

误解 3: fzf/ripgrep 可以替代当前方案
❌ 错误！它们需要访问文件系统，浏览器无法做到
```

### ✅ 正确理解

```
"后端" (构建时):
  - 运行环境：本地 Rust 程序
  - 任务：生成索引
  - 工具：jieba-rs (分词) + elasticlunr-rs (索引)
  - 输出：searchindex-xxxx.js

"前端" (运行时):
  - 运行环境：浏览器 JavaScript
  - 任务：搜索索引
  - 工具：elasticlunr.js (搜索库)
  - 输入：searchindex-xxxx.js
```

## 📦 数据流详解

### 构建时流程

```
1. 用户运行：mdbook build

2. mdbook-zh-search 处理:
   ├── 读取 book.toml 配置
   ├── 读取所有 Markdown 章节
   ├── 对每个章节:
   │   ├── 提取标题、正文、面包屑
   │   ├── 中文分词 (jieba-rs)
   │   │   └── "一句话总结" → ["一句", "句话", "一句话", "总结", ...]
   │   ├── 构建倒排索引 (elasticlunr-rs)
   │   └── 添加停用词过滤
   └── 生成文件:
       ├── searchindex-xxxx.js (索引数据)
       └── 其他静态资源 (searcher.js, elasticlunr.min.js, etc.)

3. 输出到 book/ 目录
```

**searchindex-xxxx.js 内容示例**:
```javascript
window.search = window.search || {};
window.search.index = elasticlunr.Index.load({
  "version": "0.14.0",
  "fields": ["title", "body", "breadcrumbs"],
  "fieldVectors": [
    ["0/简介", {"title": 0.5, "body": 0.3}],
    ["1/第一章", {"title": 0.6, "body": 0.4}],
    // ...
  ],
  "invertedIndex": [
    ["一句", {"_index": 0, "title": {"0": 0.5}, "body": {"1": 0.3}}],
    ["总结", {"_index": 1, "title": {"0": 0.5}, "body": {"1": 0.3}}],
    // ...
  ],
  "documentStore": {
    "docs": {
      "0": {"title": "简介", "body": "...", "breadcrumbs": "..."},
      "1": {"title": "第一章", "body": "...", "breadcrumbs": "..."},
      // ...
    }
  }
});
```

---

### 运行时流程

```
1. 用户打开书籍网站 (book/index.html)

2. 浏览器加载:
   ├── index.html
   ├── searchindex-xxxx.js (索引数据，2-5MB)
   ├── elasticlunr.min.js (搜索库，50KB)
   └── searcher.js (搜索 UI 逻辑，30KB)

3. 用户按下 'S' 或 '/' 键:
   └── 搜索框弹出

4. 用户输入搜索词 (如"总结"):
   ├── searcher.js 调用 elasticlunr.search()
   ├── elasticlunr 在索引中查找"总结"
   ├── 返回匹配的文档列表
   ├── 高亮显示关键词 (mark.js)
   └── 显示搜索结果

5. 用户点击结果:
   └── 跳转到对应页面
```

**搜索代码示例** (searcher.js):
```javascript
// 加载索引
elasticlunr.Index.load(window.search.index);

// 用户搜索
function search(query) {
    const results = searchindex.search(query, {
        bool: 'AND',
        expand: true,
        fields: {
            title: {boost: 1},
            body: {boost: 1},
            breadcrumbs: {boost: 0},
        },
    });
    
    // 显示结果
    displayResults(results);
}
```

## 🎯 架构特点

### 优势

| 特点 | 说明 | 好处 |
|------|------|------|
| **静态站点** | 无需服务器 | 部署简单，成本低 |
| **预生成索引** | 构建时完成分词和索引 | 搜索速度快 |
| **浏览器端搜索** | 无需后端 API | 完全离线可用 |
| **中文分词** | jieba-rs + trigram | 支持中文搜索 |

### 劣势

| 特点 | 说明 | 影响 |
|------|------|------|
| **索引大小** | 2-5MB | 首次加载慢 |
| **内存占用** | 10-20MB | 移动端压力 |
| **无模糊搜索** | elasticlunr 不支持 | 拼写错误找不到 |
| **无拼音搜索** | 需要额外实现 | 国际用户不便 |

## 📊 前后端对比

| 维度 | 构建时 (Rust) | 运行时 (JavaScript) |
|------|--------------|---------------------|
| **运行环境** | 本地 mdbook 程序 | 用户浏览器 |
| **时间点** | 书籍生成时 | 用户访问时 |
| **主要任务** | 分词、建索引 | 搜索、显示结果 |
| **性能要求** | 构建速度 (< 10s) | 搜索速度 (< 100ms) |
| **工具库** | jieba-rs, elasticlunr-rs | elasticlunr.js, mark.js |
| **输出/输入** | 生成 searchindex.js | 加载 searchindex.js |

## 🔧 优化方案梳理

### 方案 1：改进当前架构（推荐）

```
构建时:
  ✅ 改进分词质量 (已完成 trigram)
  ⏳ 优化停用词列表
  ⏳ 添加专业术语词典
  
运行时:
  ⏳ 改进搜索 UI
  ⏳ 添加搜索结果分页
```

**优势**:
- 架构不变，风险低
- 兼容现有书籍
- 用户无感知升级

---

### 方案 2：迁移到 Pagefind

```
构建时:
  ┌─────────────────────────────────┐
  │  mdbook-pagefind (新 crate)     │
  │                                 │
  │  1. 读取 Markdown 文件            │
  │  2. 调用 Pagefind CLI           │
  │  3. 生成 pagefind 索引           │
  └─────────────────────────────────┘
  
运行时:
  ┌─────────────────────────────────┐
  │  浏览器加载                      │
  │  ├── pagefind 索引              │
  │  ├── pagefind UI (可选)         │
  │  └── 搜索逻辑                   │
  └─────────────────────────────────┘
```

**优势**:
- 索引更小 (1-3MB vs 2-5MB)
- 搜索更快 (< 50ms)
- 默认 UI 美观
- 活跃维护

**劣势**:
- 需要新 crate
- 学习成本
- 迁移工作

---

### 方案 3：混合方案 (全文搜索 + fzf)

```
构建时:
  ├── 生成全文索引 (elasticlunr)
  └── 合并所有文本 (用于 fzf)
  
运行时:
  ├── 加载全文索引 (用于快速过滤)
  ├── 加载模糊搜索库 (Fuse.js)
  └── 混合搜索:
      1. 全文搜索快速过滤候选
      2. fzf 算法对结果排序
```

**优势**:
- 支持模糊匹配
- 支持拼写错误
- 搜索结果更智能

**劣势**:
- 索引更大 (+20%)
- 代码复杂
- 维护成本高

## 🎯 总结

### 当前架构

```
构建时 (Rust) → 生成索引 → 运行时 (JavaScript) → 搜索
     ↓                                              ↓
  分词、建索引                                    浏览器端查找
```

### 关键澄清

- ✅ **有"前后端"概念**，但都在用户侧完成
- ✅ **构建时** = 本地 Rust 程序生成索引
- ✅ **运行时** = 浏览器 JavaScript 搜索索引
- ❌ **没有服务器**，纯静态站点
