## 问题 1：为什么不能集成 fzf + ripgrep？

### ❌ 根本原因：运行环境不同

```
┌─────────────────────────────────────────────────────────┐
│  fzf / ripgrep                                          │
│                                                         │
│  运行环境：本地终端/命令行                               │
│  文件系统：直接访问                                      │
│  内存限制：无（可用数 GB）                               │
│  生命周期：用完即走                                      │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  mdbook 搜索                                             │
│                                                         │
│  运行环境：浏览器                                        │
│  文件系统：❌ 无法访问                                   │
│  内存限制：严格（200-500MB 上限）                        │
│  生命周期：持续运行（用户会多次使用）                    │
└─────────────────────────────────────────────────────────┘
```

---

### 🔍 技术障碍详解

#### 障碍 1：文件系统访问

```javascript
// ripgrep 需要这样访问文件
const files = await fs.readdir('./book/');
for (const file of files) {
    const content = await fs.readFile(file);
    // ... 搜索
}

// ❌ 浏览器中：
// - 无法访问本地文件系统
// - 只能加载已生成的文件（如 searchindex.js）
// - 无法动态读取 markdown 源文件
```

#### 障碍 2：内存限制

```
ripgrep 内存使用（5k 行文本）:
├── 文件内容加载：700KB
├── 正则引擎：5-10MB
├── 匹配结果缓存：10-20MB
└── 总计：~20-30MB ✅ 可行

ripgrep 内存使用（50k 行文本）:
├── 文件内容加载：7MB
├── 正则引擎：10-20MB
├── 匹配结果缓存：100-200MB
└── 总计：~120-230MB ⚠️ 危险

ripgrep 内存使用（200k 行文本）:
├── 文件内容加载：28MB
├── 正则引擎：20-50MB
├── 匹配结果缓存：400-800MB
└── 总计：~450-880MB ❌ 浏览器崩溃
```

**对比全文搜索**：
```
elasticlunr 内存使用（200k 行文本）:
├── 索引加载：40MB
├── 搜索算法：< 5MB
└── 总计：~45MB ✅ 安全
```

#### 障碍 3：性能问题

```
ripgrep 搜索流程:
1. 读取所有文件 → 100-500ms
2. 逐行扫描匹配 → 200-1000ms
3. 返回结果 → 总计 300-1500ms

elasticlunr 搜索流程:
1. 索引查找（O(1)） → < 50ms
2. 返回结果 → 总计 < 50ms

差距：6-30 倍
```

---

### 🧪 实际测试：ripgrep-wasm

有人尝试过将 ripgrep 编译为 WebAssembly：

**项目**: https://github.com/ripgrep-wasm/ripgrep-wasm

**结果**：
```
✅ 技术可行
❌ 实际不可用

原因:
- WASM 模块大小：10MB+
- 需要加载所有源文件：50-200MB
- 首次搜索时间：500-2000ms
- 移动端：内存超限崩溃
```

---

### ✅ 替代方案：用 JavaScript 实现类似功能

```
fzf 的核心功能 → Fuse.js（模糊搜索）
ripgrep 的核心功能 → 原生 RegExp（正则搜索）

优势:
- 专为浏览器设计
- 内存占用低
- 性能可接受
- 无需 WASM
```

#### 2. JavaScript 实现

```javascript
// 搜索模式枚举
const SearchMode = {
    EXACT: 'exact',
    REGEX: 'regex',
    FUZZY: 'fuzzy',
};

// 搜索状态
const searchState = {
    mode: SearchMode.EXACT,
    caseSensitive: false,
    index: null,
    fuse: null,
};

// 初始化
function initSearch() {
    // 加载索引
    searchState.index = elasticlunr.Index.load(window.search.index);
    
    // 构建 Fuse 索引（用于模糊搜索）
    const documents = Object.values(
        searchState.index.documentStore.docs
    ).map((doc, id) => ({
        id,
        title: doc.title,
        body: doc.body,
        breadcrumbs: doc.breadcrumbs,
    }));
    
    searchState.fuse = new Fuse(documents, {
        keys: [
            { name: 'title', weight: 2 },
            { name: 'body', weight: 1 },
        ],
        threshold: 0.3,
        distance: 100,
        isCaseSensitive: false,
        includeMatches: true,
    });
    
    // 绑定事件
    bindEvents();
}

// 绑定事件
function bindEvents() {
    const searchInput = document.getElementById('search-input');
    const btnMode = document.getElementById('btn-mode');
    const btnCase = document.getElementById('btn-case');
    
    // 搜索输入
    searchInput.addEventListener('input', (e) => {
        const query = e.target.value;
        performSearch(query);
    });
    
    // 切换模式
    btnMode.addEventListener('click', () => {
        const modes = [
            SearchMode.EXACT,
            SearchMode.REGEX,
            SearchMode.FUZZY,
        ];
        
        const currentIndex = modes.indexOf(searchState.mode);
        searchState.mode = modes[(currentIndex + 1) % modes.length];
        
        updateModeButton();
        performSearch(searchInput.value);
    });
    
    // 切换大小写
    btnCase.addEventListener('click', () => {
        searchState.caseSensitive = !searchState.caseSensitive;
        updateCaseButton();
        performSearch(searchInput.value);
    });
    
    // 键盘快捷键
    document.addEventListener('keydown', (e) => {
        if (e.key === '/' && !isInputFocused()) {
            e.preventDefault();
            searchInput.focus();
        }
    });
}

// 执行搜索
function performSearch(query) {
    if (!query) {
        hideResults();
        return;
    }
    
    let results;
    
    switch (searchState.mode) {
        case SearchMode.REGEX:
            results = regexSearch(query, searchState.caseSensitive);
            break;
        case SearchMode.FUZZY:
            results = fuzzySearch(query, searchState.caseSensitive);
            break;
        default:
            results = exactSearch(query, searchState.caseSensitive);
    }
    
    displayResults(results);
    updateHint(query);
}

// 精确搜索
function exactSearch(query, caseSensitive) {
    return searchState.index.search(query, {
        bool: 'AND',
        expand: true,
        fields: {
            title: { boost: 2 },
            body: { boost: 1 },
        },
    });
}

// 模糊搜索
function fuzzySearch(query, caseSensitive) {
    // 更新 Fuse 配置
    searchState.fuse.options.isCaseSensitive = caseSensitive;
    
    const results = searchState.fuse.search(query);
    
    return results.map(r => ({
        ref: r.item.id,
        score: r.score,
        doc: searchState.index.documentStore.docs[r.item.id],
        matches: r.matches,
    }));
}

// 正则搜索
function regexSearch(query, caseSensitive) {
    try {
        const flags = caseSensitive ? 'g' : 'gi';
        const regex = new RegExp(query, flags);
        
        const docs = searchState.index.documentStore.docs;
        const results = [];
        
        for (const [ref, doc] of Object.entries(docs)) {
            const allText = `${doc.title} ${doc.body}`;
            const matches = allText.match(regex);
            
            if (matches) {
                results.push({
                    ref,
                    score: matches.length,
                    doc,
                    matches: matches.slice(0, 5),
                });
            }
        }
        
        return results.sort((a, b) => b.score - a.score);
    } catch (e) {
        console.error('Invalid regex:', e);
        return [];
    }
}

// 更新模式按钮
function updateModeButton() {
    const btn = document.getElementById('btn-mode');
    const labels = {
        [SearchMode.EXACT]: '🔍 精确',
        [SearchMode.REGEX]: '.* 正则',
        [SearchMode.FUZZY]: '✨ 模糊',
    };
    
    btn.textContent = labels[searchState.mode];
    btn.title = `当前模式：${labels[searchState.mode]}`;
}

// 更新大小写按钮
function updateCaseButton() {
    const btn = document.getElementById('btn-case');
    btn.classList.toggle('active', searchState.caseSensitive);
    btn.title = searchState.caseSensitive 
        ? '大小写敏感 (开启)' 
        : '大小写敏感 (关闭)';
}

// 显示搜索提示
function updateHint(query) {
    const hint = document.getElementById('search-hint');
    
    const hints = {
        [SearchMode.EXACT]: '🔍 精确匹配模式',
        [SearchMode.REGEX]: '.* 正则表达式模式',
        [SearchMode.FUZZY]: '✨ 模糊搜索模式 (支持拼写错误)',
    };
    
    hint.textContent = hints[searchState.mode];
}
```

## 📊 配置选项对比

### VSCode 风格

```
[搜索框] [🔍] [.*] [Aa] [✨]

优点:
✅ 直观，一目了然
✅ 快速切换
✅ 专业感强

缺点:
❌ 占用空间
❌ 按钮多可能让用户困惑
```

### 下拉菜单风格

```
[搜索框] [模式：精确 ▼]

优点:
✅ 节省空间
✅ 清晰列出所有选项

缺点:
❌ 需要额外点击
❌ 不够直观
```

### 智能检测 + 手动覆盖

```
[搜索框]
自动检测：模糊搜索 [切换]

优点:
✅ 默认智能，省心
✅ 允许手动覆盖

缺点:
❌ 用户可能不知道可以切换
```
