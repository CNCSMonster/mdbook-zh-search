# mdBook 中文搜索支持方案

## 1. 背景与问题分析

### 1.1 当前 mdBook 搜索机制

mdBook 的搜索功能基于以下技术栈：
- **构建时**: 使用 Rust 实现的 `elasticlunr-rs` 库生成搜索索引
- **运行时**: 使用 JavaScript 实现的 `elasticlunr.js` 在浏览器端执行搜索
- **分词方式**: 默认按空格和连字符分割文本

### 1.2 中文搜索失败的根本原因

中文搜索无法工作的核心问题是**分词（Tokenization）**：

| 语言类型 | 词边界 | 分词方法 |
|---------|--------|---------|
| 拉丁语系（英语等） | 显式空格 | 简单的空格分割 |
| CJK（中文、日文、韩文） | **无显式边界** | 需要专门的分词算法 |

**技术问题详解：**

1. **默认分词器失效**: `elasticlunr` 默认按空格分割文本。中文文本词与词之间没有空格，导致整句话被当作一个"token"。

2. **中文分词复杂性**: 中文分词需要：
   - 统计决策
   - 大型词典
   - 句法和语义分析
   - 例如歧义处理："美国会不同意" 可以是 "美国/会/不同意" 或 "美/国会/不同意"

3. **词干提取需求**: 不同语言需要特定的词干提取器来匹配词形变化

### 1.3 现有解决方案调研

| 项目 | 方法 | 中文分词 | 维护状态 | 可用性 |
|-----|------|---------|---------|-------|
| mdbook-search-chinese | elasticlunr-rs + 停用词 | ❌ 无 | ❌ 2021 年后停止 | 部分 |
| lunr-languages | 自定义分词器 | ✅ 是 | ✅ 活跃 | ✅ 可用 (JS) |
| FlexSearch | 内置 CJK 编码器 | ✅ 是 | ✅ 活跃 | ✅ 可用 (JS) |
| tantivy+jieba | jieba-rs 分词 | ✅ 是 | ✅ 活跃 | 过于重量级 |

**关键发现**: 现有的 `mdbook-search-chinese` 分支仅添加了中文停用词，但缺乏真正的词分词功能。

## 2. 解决方案设计

### 2.1 方案概述

本方案采用**双层分词策略**：
1. **构建时分词**: 修改 `elasticlunr-rs` 索引生成，使用 `jieba-rs` 进行中文分词
2. **运行时搜索**: 保持 `elasticlunr.js`，但改进前端分词逻辑

### 2.3 详细实现方案

#### 方案 A: 修改 elasticlunr-rs（推荐）

**优点**:
- 保持与现有 mdBook 架构兼容
- 构建时完成分词，运行时性能好
- 索引质量高

**缺点**:
- 需要修改 Rust 代码
- 增加构建依赖（jieba-rs）

**实现步骤**:

1. **修改 `mdbook-html/src/html_handlebars/search.rs`**:
   ```rust
   // 添加 jieba-rs 依赖
   use jieba_rs::Jieba;
   
   // 创建全局 jieba 实例
   lazy_static::lazy_static! {
       static ref JIEBA: Jieba = Jieba::new();
   }
   
   // 改进的分词函数
   fn tokenize(text: &str) -> Vec<String> {
       // 检测是否为中文文本
       if is_cjk_text(text) {
           // 使用 jieba 分词
           JIEBA.cut_all(text)
               .filter(|s| !s.is_empty() && s.len() <= MAX_WORD_LENGTH_TO_INDEX)
               .map(|s| s.to_lowercase())
               .collect()
       } else {
           // 保持原有英文分词逻辑
           text.split(|c: char| c.is_whitespace() || c == '-')
               .filter(|s| !s.is_empty())
               .map(|s| s.trim().to_lowercase())
               .filter(|s| s.len() <= MAX_WORD_LENGTH_TO_INDEX)
               .collect()
       }
   }
   
   // 检测 CJK 文本
   fn is_cjk_text(text: &str) -> bool {
       text.chars().any(|c| {
           matches!(c,
               '\u{4e00}'..='\u{9fff}' |  // CJK Unified Ideographs
               '\u{3400}'..='\u{4dbf}' |  // CJK Extension A
               '\u{f900}'..='\u{faff}' |  // CJK Compatibility Ideographs
               '\u{3040}'..='\u{30ff}' |  // Hiragana/Katakana
               '\u{ac00}'..='\u{d7af}'    // Hangul Syllables
           )
       })
   }
   ```

2. **修改 `Cargo.toml` 添加依赖**:
   ```toml
   [dependencies]
   jieba-rs = "0.6"
   lazy_static = "1.4"
   ```

3. **配置支持**:
   ```toml
   [output.html.search]
   enable = true
   language = "zh"  # 新增配置项
   ```

#### 方案 B: 集成 lunr-languages（备选）

**优点**:
- 成熟的 JavaScript 解决方案
- 支持多种语言

**缺点**:
- 需要替换 elasticlunr.js
- 索引生成逻辑仍需修改

**实现步骤**:

1. 下载 `lunr-languages` 的中文支持文件
2. 替换 `elasticlunr.min.js` 为 `lunr.zh.js`
3. 修改 `searcher.js` 以适配 lunr.js API

#### 方案 C: 使用 FlexSearch（激进方案）

**优点**:
- 现代搜索引擎
- 原生支持 CJK
- 性能优秀

**缺点**:
- 需要完全重写搜索模块
- 索引格式不兼容

## 4. 技术细节

### 4.1 jieba-rs 分词模式选择

```rust
// 三种分词模式：
// 1. cut() - 精确模式，最适合文本搜索
// 2. cut_all() - 全模式，适合索引构建
// 3. cut_for_search() - 搜索引擎模式，基于精确模式

// 推荐使用 cut_for_search() 用于搜索索引
let words = JIEBA.cut_for_search(text, false);
```

### 4.2 索引大小优化

中文分词会产生更多 token，需要优化：

1. **停用词过滤**: 过滤常见无意义词（的、了、是等）
2. **词长限制**: 限制单个 token 最大长度
3. **词频过滤**: 过滤出现频率过低的词

### 4.3 性能考虑

```rust
// 使用懒加载初始化 jieba
lazy_static::lazy_static! {
    static ref JIEBA: Jieba = Jieba::new();
}

// 或者使用 OnceCell (Rust 1.70+)
use std::sync::OnceLock;

static JIEBA: OnceLock<Jieba> = OnceLock::new();

fn get_jieba() -> &'static Jieba {
    JIEBA.get_or_init(Jieba::new)
}
```
